### 8.2.10. Seguridad en Windows
#### Mecanismos de seguridad
#### Estructuras de datos utilizadas para implementar la seguridad
- Ficha de acceso (access token)
	- Asignada a primer proceso creado para el usuario (explorer.exe) y heredada por todos los procesos creados a partir de este
	- Datos:
		- Identificador de seguridad del usuario al que pertenece el proceso (SID)
		- Identificador de seguridad del grupo de usuarios al que pertenece el proceso
		- Grupos a que pertenece el usuario (necesario para API POSIX)
		- Lista de control de acceso discrecional (DACL)
			- La DACL contiene una o varias entradas con:
				- SID (de usuario o grupo)
				- Estado (permitir o denegar)
				- Conjunto de operaciones seleccionadas sobre el objeto
		- Privilegios 
- Descriptor de seguridad. 
	- Estructura de datos que especifica las operaciones que pueden realizar los diferentes usuarios sobre el objeto
	- Datos
		- SID del usuario propietario del objeto
		- SID del grupo propietario del objeto
		- Puntero a la lista de acceso discrecional (DACL)
